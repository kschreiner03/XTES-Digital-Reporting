1 PURPOSE

The purpose of this procedure is to provide detailed instructions on how to use the X-TES Digital Reporting Application for the purpose of data collection. This procedure applies to all X-Terra biophysical staff utilizing the application for data collection.

2 SCOPE

This procedure outlines the steps required to utilize the X-TES Digital Reporting Application to collect data and information. The details provided in this document are sufficient to train any X-Terra staff member to perform this task competently.

3 GENERAL PROTOCOL

The X-TES Digital Reporting Application is a desktop program that must be installed from the company server. Once installed, it allows users to create reports from templates. Data is entered into specific fields, and prepared photos are uploaded into the photographic log sections. The application saves work-in-progress as editable project files (with extensions .plog, .dfr, or .spdfr).

It is critical to distinguish between saving a project file (for future editing) and generating a final PDF (for submission). All recent work is stored locally on the user's computer for quick access from the application's landing page. The most critical workflow component is the mandatory preparation of photos (cropping, resizing, and server archival) before they are used in a report, as this ensures quality and manageable file sizes.

4 DETAILED PROCEDURES

4.1 Installation & Updates

This section covers the first-time installation and subsequent updating of the application on a Windows computer.

  1. Access Installer Files: An active connection to the company network (in-office or via VPN) is required. Open File Explorer (shortcut: Windows Key + E) and navigate to the server path: Z:\Templates\XTEC Apps\x64. This folder contains the most current version.
  2. Copy Files to Local Drive:
    CRITICAL: Do not run the installer directly from the server. Network permissions will cause the installation to fail. You MUST copy the files to your local computer first.
    Create a new, empty folder on your Desktop (e.g., "XTEC_Installer"). From the server folder, copy the three required files:
    • A file named exactly RELEASES.
    • A large file with a .nupkg extension (e.g., x-tec-digital-reporting-web-1.0.2-full.nupkg).
    • The application executable: X-TES Digital Reporting.exe.
    Paste these three files into the "XTEC_Installer" folder on your Desktop.
  3. Run the Installer: Open the "XTEC_Installer" folder on your Desktop and double-click the X-TES Digital Reporting.exe file. The installation will proceed automatically with no prompts.
    Note on Windows SmartScreen: The first time you run the installer, a blue Windows Defender SmartScreen window may appear. This is normal. Click "More info," then click the "Run anyway" button that appears.
    Once complete, the application will launch, and shortcuts will be placed on your Desktop and in the Start Menu. You may now safely delete the "XTEC_Installer" folder.
  4. Updating the Application: The process for updating is identical to the first-time installation. Simply repeat the steps above with the new files from the server. The installer will automatically overwrite the old version while preserving all your locally stored project data.

4.2 Photo Preparation (CRITICAL)

THIS IS THE SINGLE MOST IMPORTANT WORKFLOW. Failure to follow these steps results in inconsistent report quality, excessively large PDF files that cannot be emailed, and incorrect project archiving.
  1. Crop to 4:3 Aspect Ratio: The final PDF report formats all images into a standard landscape 4:3 aspect ratio. The application will automatically fit images, often resulting in undesirable letterboxing (white bars). To ensure your photos are not awkwardly cut off or displayed with bars, you must crop them manually first.
    • Open a site photo in a photo editor (Windows Photos is sufficient). Find the "Crop" function, select "Aspect Ratio," and choose 4:3. Adjust the cropping rectangle to perfectly frame the subject matter, then save. It's recommended to save it as a new copy to preserve the original.
  2. Resize for Optimal File Size: A photo from a modern phone can be over 5MB. A report with 20 such photos would result in an unusable 100MB+ PDF. You must create smaller copies for the report.
    • Using your photo editor's "Resize" function on the cropped images, set the width to 1024 pixels. The height should adjust automatically to 768 pixels. This resolution is perfect for high-quality printing and on-screen viewing while keeping file sizes small.
  3. Server File Management: Proper archival is the final mandatory step.
    • Navigate to the appropriate project folder on the X-Terra server.
    • Within that folder, create two sub-folders: Photos - Originals and Photos - For Reporting.
    • Upload your original, full-resolution photos into the "Photos - Originals" folder.
    • Upload your cropped and resized (1024px wide) photos into the "Photos - For Reporting" folder. These are the files you will actually use in the application.

4.3 User Interface Overview

4.3.1 The Landing Page

This is the main hub of the application.

  • Create a New Report: The three large cards at the top are for starting a new, blank project.
  • Recent Projects: This section lists projects you have saved. This data is stored locally on your computer for quick access. Use the search bar to filter this list by project name or number.

4.3.2 The Report Creation Screen

This is where you will do all your work.

  • Top Bar: Contains a "Home" button to return to the Landing Page (unsaved work will be lost) and a group of action buttons on the right for "Open Project," "Save Project," and "Save to PDF."
  • Desktop Menu Bar: In the installed desktop app, a menu bar at the very top of the window contains:
    • File > Download Photos: Exports all photos and metadata into a zip file.
    • Help > How to Use: Opens this SOP document.
    • Help > Report Issues: Opens a web form to report bugs or suggest improvements.
  • Symbols Palette: A floating vertical tab on the right edge of the screen. Clicking it opens a palette of special characters (°, ±, µ). Clicking any character copies it to your clipboard, ready to be pasted.

4.4 Report Creation & Data Entry

CRITICAL DATE FORMAT: In all report headers, the date must be entered in the format Month Day, Year (e.g., October 1, 2025). Using other formats (e.g., 10/01/2025) may cause errors when generating filenames or displaying dates.

4.4.1 Photographic Log

The simplest report type. Fill out all header fields (Proponent, Location, Date, Project #, Project Name) and proceed to add photos as described in section 4.5.

4.4.2 Standard Daily Field Report (DFR)

A comprehensive report for most projects, combining detailed text sections with an integrated photographic log.

  1. Header Fields: Fill out all required fields. The dropdown field (defaulting to "MOE FILE #") allows you to specify the type of environmental file number being referenced.
  2. Using Location-Specific Activities: The Project Activities section is for the main narrative. It is broken into two types of entries:
    • General Activity: Use this large text box for the main overview of the day or for activities not tied to one specific location (e.g., travel, tailgate meetings, general site inspections).
    • Location-Specific Activity (Optional): This is a powerful tool for projects with work at multiple, distinct locations in one day. Click "Add Location-Specific Activity" to create a separate, organized block for each location. For example, you can create one block for "Wetland A" and detail all work performed there, then a second block for "Wetland B." This creates a much clearer report than a single long narrative.
  3. Other Text Sections & Bullet-Point Editor: The multi-line text fields (Communication, Weather, etc.) use a custom editor.
    • Press Enter to create a new bullet point on the next line.
    • Press Tab to indent the current line (create a sub-bullet).
    • Press Shift + Tab to outdent the current line.
    • Press Enter on an empty bulleted line to outdent or create a normal paragraph break.

4.4.3 SaskPower Daily Field Report (DFR)

A specialized DFR tailored for SaskPower projects with unique fields and a safety checklist.

  1. Report Information: Fill in all fields, including the unique fields for Vendor & Foreman, ENV File Number, and Total Hours Worked.
  2. Safety Checklist: Select "Yes," "No," or "NA" for each of the three safety-related items.
  3. Project Activities: Unlike the standard DFR, there is only one main text box for all activities. If work occurred at multiple locations, they must be described sequentially within this single box. A detailed, timestamped log is required.
    4. Note on Opening Older Files: If you open an older SaskPower DFR file that used separate location blocks, the application will automatically merge all activity text into the single "Project Activities" box to conform to the new standard. A notice will appear. Be sure to review and save the project to finalize this change.

4.5 Photographic Log Management

  1. Adding Entries: Click "Add Photo" for a standard entry. In the Standard DFR, you can also click "Add Map" for overview images like GPS tracks or site plans. Maps are given their own full page in the final PDF for better readability and are automatically labeled sequentially (Map 1, Map 2, etc.). You can also click "Add Photo Here" between existing entries to insert a new photo at a specific position.
  2. Uploading Images: Click the camera icon area and select one of your prepared, cropped and resized images.
  3. Managing Entries:
    • Use the up and down arrows to reorder photos. All photo/map numbers will update automatically.
    • Click the trash can icon to permanently delete an entry.
    • Hover over an uploaded image and click the "enlarge" icon that appears in the top-right corner to view it full-screen.

4.6 Saving, Exporting & Data Management

THE GOLDEN RULE: "SAVE PROJECT" VS. "SAVE TO PDF". Understanding this distinction is essential to avoid data loss and rework.
  • Save Project: Creates a live, EDITABLE source file (.dfr, .plog). This is your master copy for saving progress and making future edits. You can only open these files with this application.
  • Save to PDF: Generates a final, NON-EDITABLE document for submission, printing, or sharing. It cannot be opened or edited by this application.
  1. Procedure: Saving a Project File: Click the "Save Project" button. A "Save As" dialog will appear. Choose a location on your computer to save the file. This action also automatically adds or updates the project in your "Recent Projects" list for quick access.
  2. Procedure: Generating a PDF:
    Best Practice: Always click "Save Project" immediately before you "Save to PDF." This ensures you have an editable source file that perfectly matches the final PDF.
    Ensure all required fields (highlighted in red if empty) are filled. Click "Save to PDF." A full-screen preview will appear. Review it carefully. If it is correct, click the "Download PDF" button to save the final file.
  3. Procedure: Downloading All Photos (Desktop App): In the desktop app menu, go to File > Download Photos. The application will generate and ask you to save a single .zip file. This file contains all your report images (named by photo/map number) and a metadata.txt file with all associated text (date, location, direction, description).

4.7 Advanced Features & Data Concepts

  1. File Associations ("Open with..."): You can set the application as the default program for project files to open them with a double-click. Right-click a project file (e.g., my_project.dfr), choose "Open with > Choose another app," and navigate to the application shortcut on your Desktop. Select the X-TES Digital Reporting shortcut file and check "Always use this app...". Repeat for .plog and .spdfr files.
  2. Understanding Local Data Storage: The "Recent Projects" list is powered by a local database (IndexedDB) on your computer. This allows for instant loading of your recent work. This data is stored only on the computer you are using and does not sync to the cloud or other machines.
  3. Removing vs. Deleting Recent Projects: On the landing page, clicking the "..." on a recent project gives two options:
    • Remove from recent: A safe action that only removes the project's shortcut from the landing page list. Your saved project file (.dfr) on your computer is unaffected.
    • Delete project permanently: This removes the temporary saved file from the application's local database. You will need to open your saved project file from the server following this step.

5 FORMATTING TIPS

Here are some formatting tips to assist with your final report.

Issue: Hanging header (aka – text block is on separate page from header)

Fix: Add blank spaces in the text box above the section header you want to push to the next page.
